Filter
Filter类为一种操作，继承自Operator类，而Operator类又实现了OpIterator接口
因此，Filter的本质上即为一个迭代器，根据Predicate中的filter方法对所有的tuple进行过滤，再使用迭代器来迭代过滤后的结果
在Operator类中，存在一个fetchNext的抽象方法，并且在next方法中进行调用
fetchNext为根据该操作的逻辑获取下一个Tuple（可能是filter，可以能是Join，具体逻辑由继承的子类来定义实现）
next()即为一般迭代器中的next，用于和hasNext()配合迭代

@Override

Insert

static class HeapFileDuplicates extends HeapFile {

    	private final int duplicates;
  
    	public HeapFileDuplicates(File f, TupleDesc td, int duplicates) {
    		super(f, td);
    		this.duplicates = duplicates;
    	}
  
    	// this version of insertTuple inserts duplicate copies of the same tuple,
    	// each on a new page
    	@Override
    	public List<Page> insertTuple(TransactionId tid, Tuple t)
    			throws DbException, IOException {
    		List<Page> dirtypages = new ArrayList<>();
    		for(int i = 0; i < duplicates; i++) {
    			// create a blank page
    			BufferedOutputStream bw = new BufferedOutputStream(new FileOutputStream(super.getFile(), true));
                byte[] emptyData = HeapPage.createEmptyPageData();
                bw.write(emptyData);
                bw.close();
    			HeapPage p = new HeapPage(new HeapPageId(super.getId(), super.numPages() - 1),
    					HeapPage.createEmptyPageData());
    	        p.insertTuple(t);
    			dirtypages.add(p);
    		}
    		return dirtypages;
    	}
    }

buffer pool重构

之前buffer pool采用的策略为一个concurrentHashMap来作为内存中的页管理，在额外添加一个LinkedList来实现FIFO的页淘汰策略，需要进行淘汰时则从链表的头部进行淘汰

list起到的是一个索引的作用，只是告诉buffer pool应该淘汰哪一页，存在一定的额外空间占用，并且实现的也只是FIFO级别的页淘汰策略，命中率并不高，在后续lab4时需要进行页淘汰时，这种额外一个list进行管理的结构操作起来也越来越麻烦，因此选择进行重构

在物理层面上，只使用一个concurrentHashMap作为页管理，内部存放节点，节点之间再通过链表的形式串联起来

首先定义节点

提供一些方法用于进行lru时对链表进行内存管理

LinkedNode head;
LinkedNode tail;

private void addToHead(LinkedNode node) {
    node.prev = head;
    node.next = head.next;
    head.next.prev = node;
    head.next = node;

}

private void remove(LinkedNode node) {
    node.prev.next = node.next;
    node.next.prev = node.prev;
}

private void moveToHead(LinkedNode node) {
    remove(node);
    addToHead(node);
}

private LinkedNode removeTail() {
    LinkedNode node = tail.prev;
    remove(node);
    return node;
}
  • 定义一个dummyHead和一个dummyTail，不存储真实数据，仅用于标识
  • 链表的头部维护最新的page，如果buffer pool当中的page被访问到了，则将其移动到链表的头部位置
  • 链表的尾部标识长时间未被访问的节点，淘汰则从尾部开始进行淘汰

最终在内存淘汰策略上，实现的为一种近似LRU的策略，相比于直接FIFO，效果还是要好上许多
