Exercise1

选择率的计算

参考 实验指导中给出的公式即可，在此处简单梳理一下：
首先对于选择率，根据定义来看即为符合条件的tuple占所有的tuple的比例，通过直方图来表示后，即求符合条件的面积占总面积的比例即可
首先给出的假设是在一个bucket当中，tuple的数值为均匀分布，即如果[0,10)中有五个数据，则为0,2,4,6,8
因此，直方柱中所有的tuple即为该直方柱的值，直接累加即可，重点为被分隔开的直方柱中的tuple数量
被分隔开的直方柱中的tuple数量 = 比例 * 总数，比例通过宽度比来求，转换为代码即为：

double bPart = (v - (min + index * width)) / width;
double bFraction = selectBuckets[index] * 1.0 ;

最后在使用求出的符合条件的tuple数 / 总tuple数即可

tuple += bPart * bFraction;
return tuple / ntups;

参数

  • buckets：整个统计直方图中bucket的数量
  • min/max：上下限
  • width：单个bucket的宽度
  • ntups：整张表中tuple的数量
  • selectBuckets：直方图

其中，width的计算公式为:width = (max - min +1.0) / buckets;index的起始值为0，通过+1.0的方式来保证最后一个值不会超限

实现方法：

自定义一个getIndex()，用于找到新增的值最终属于哪个bucket

private int getIndex(int value) {
        return (int) ((value - min) / width);
    }

对于不同选择率的计算，可以只实现一个 < 后续所有的都通过操作符之间的关系，在<的基础之上推导而来即可

public double estimateSelectivity(Predicate.Op op, int v) {
        switch (op) {
            case LESS_THAN:
                if (v <= this.min) {
                    return 0.0;
                } else if (v >= this.max) {
                    return 1.0;
                } else {
                    int index = getIndex(v);
                    double tuple = 0;
                    for(int i = 0; i < index; i++) {
                        tuple += selectBuckets[i];
                    }
                    double bPart = v - (min + index * width);
                    double bFraction = selectBuckets[index] / width;
                    tuple += bPart * bFraction;
                    return  tuple / ntups;
                }
            case GREATER_THAN:
                return 1- estimateSelectivity(Predicate.Op.LESS_THAN_OR_EQ,v);
            case GREATER_THAN_OR_EQ:
                return estimateSelectivity(Predicate.Op.GREATER_THAN,v-1);
            case LESS_THAN_OR_EQ:
                return estimateSelectivity(Predicate.Op.LESS_THAN,v + 1);
            case EQUALS:
                return estimateSelectivity(Predicate.Op.LESS_THAN_OR_EQ,v) - estimateSelectivity(Predicate.Op.LESS_THAN,v);
            case NOT_EQUALS:
                return 1 - estimateSelectivity(Predicate.Op.EQUALS,v);

            default:
                throw new IllegalArgumentException("Operation is illegal");
        }

Exercise2

实现TableStats,对IntHistagram和StringHistagram的封装，对一张表的开销进行估计
计算量：

  • ScanCost：遍历整张表的开销，pageNum * ioCostPerNum
  • 选择基数：选择率乘总的tuple数selectivityFactor * tupleNum;
  • 平均选择率：判断类型后调用Histgram的封装即可
  • 估计选择率：判断类型调用Histgram的封装

计算参数：

   private int pageNum;
   private int tupleNum;
   private int fieldNum;
private int ioCostPerPage;

主要的工作通过构造方法来遍历整张表来获取用于计算的这些数据,封装到Histgram当中去
遍历每个field，再遍历整张表，统计出该field的一些元数据（max，min）用于构建直方图，
对每一个field都构建一个直方图，通过hashMap来分别管理Int类型和String类型的
最后再对Histgram填充数据，addValue

static final int NUM_HIST_BINS = 100;
    private DbFile dbFile;
    private int ioCostPerPage;
    private TupleDesc tupleDesc;
    private int tableId;
    private int pageNum;
    private int tupleNum;
    private int fieldNum;
    Map<Integer,IntHistogram> intHistogramMap;
    Map<Integer,StringHistogram> stringHistogramMap;

   
    public TableStats(int tableid, int ioCostPerPage) {
        this.tableId = tableid;
        this.ioCostPerPage = ioCostPerPage;
        this.dbFile = Database.getCatalog().getDatabaseFile(tableid);
        this.tupleDesc = dbFile.getTupleDesc();
        HeapFile heapFile = (HeapFile) dbFile;
        this.pageNum = heapFile.numPages();
        intHistogramMap = new HashMap<>();
        stringHistogramMap = new HashMap<>();
        Type[] type = getType(tupleDesc);
        this.fieldNum = type.length;
        this.tupleNum = 0;
        TransactionId transactionId = new TransactionId();
        SeqScan seqScan = new SeqScan(transactionId,tableId);
        int[] maxs = new int[fieldNum];
        int[] mins = new int[fieldNum];
        try {
            seqScan.open();

            for (int i = 0;i < fieldNum;i++) {

                int max = Integer.MIN_VALUE;
                int min = Integer.MAX_VALUE;
                while (seqScan.hasNext()) {
                    if(i == 0) {
                        tupleNum++;
                    }
                    Tuple tuple = seqScan.next();
                    Field field = tuple.getField(i);
                    Type type1 = field.getType();
                    if (type1 == Type.STRING_TYPE) {
                        continue;
                    }
                    IntField intField = (IntField) field;
                    int val = intField.getValue();
                    max = Math.max(max,val);
                    min = Math.min(min,val);

                }
                maxs[i] = max;
                mins[i] = min;
                seqScan.rewind();
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            seqScan.close();
        }
        for (int i = 0; i < fieldNum;i++) {
            if (type[i] == Type.INT_TYPE) {
                IntHistogram intHistogram = new IntHistogram(NUM_HIST_BINS,mins[i],maxs[i]);
                intHistogramMap.put(i,intHistogram);
            } else {
                StringHistogram stringHistogram = new StringHistogram(NUM_HIST_BINS);
                stringHistogramMap.put(i,stringHistogram);
            }
        }

        addValueToHist();

    }

    private Type[] getType(TupleDesc tupleDesc) {
        int numFields = tupleDesc.numFields();
        Type[] types = new Type[numFields];
        for (int i = 0; i <numFields;i++) {
            types[i] = tupleDesc.getFieldType(i);
        }
        return types;

    }
    private void addValueToHist(){
        TransactionId tid = new TransactionId();
        SeqScan seqScan = new SeqScan(tid,tableId);
        try {
            seqScan.open();
            while (seqScan.hasNext()) {
                Tuple tuple = seqScan.next();
                for (int i = 0; i < fieldNum;i++) {
                    Field field = tuple.getField(i);
                    Type type = field.getType();
                    if (type == Type.INT_TYPE) {
                        IntField intField = (IntField) field;
                        int value = intField.getValue();
                        intHistogramMap.get(i).addValue(value);
                    } else {
                        StringField stringField = (StringField) field;
                        String value = stringField.getValue();
                        stringHistogramMap.get(i).addValue(value);
                    }
                }

            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            seqScan.close();
        }

    }

Exercise3

实现 joinOptimizer中的estimateJoinCost()和estimateJoinCardnality方法
estimateJoinCost()
此处采用的 join算法为nested-Loop-Join即为双层嵌套循环，但是需要考虑从磁盘中的读取的开销，此处提供的参数为：

  • card1：table1的选择基数，即table1中读取的tuple总数
  • card2：table2的选择基数
  • cost1：SeqScan table1的开销
  • cost2：SeqScan table2的开销

因此总的开销为读取第一个page的开销，然后内部嵌套读取table2的开销，以及最终CPU计算m * n的开销

return cost1 + card1 * cost2 + card1 * card2;

estimateJoinCardnality()
按照实验指导中的规则进行计算，但是规则的合理性暂时还没太搞懂究竟取了何种近似计算

  • 如果join符号为 = :

    • 如果一个join的字段为主键，则基数为另一张表的基数
    • 如果两个都不是主键，则取二者中最大的
    • 如果两个都是主键，则取最小的

  • 如果为范围比较：30% * card1 * card2

public static int estimateTableJoinCardinality(Predicate.Op joinOp,
                                                  String table1Alias, String table2Alias, String field1PureName,
                                                  String field2PureName, int card1, int card2, boolean t1pkey,
                                                  boolean t2pkey, Map<String, TableStats> stats,
                                                  Map<String, Integer> tableAliasToId) {
       int card = 1;

       if (joinOp == Predicate.Op.EQUALS) {
           if (t1pkey && !t2pkey) {
               card = card2;
           } else if (!t1pkey && t2pkey) {
               card = card1;
           } else if (!t1pkey && !t2pkey) {
               card = Math.max(card1,card2);
           } else {
               card = Math.min(card1,card2);
           }
       } else {
           card = (int)(0.3 * card1 * card2);
       }
       return card <= 0 ? 1 : card;

最开始看给了那么多参数，还以为要通过stats拿到对应的直方图，从其中获取到card,后来发现直接通过传入的值实现即可

  • computeCostAndCardOfSubplan()用于计算出给定一个子集合和子集合中的一个join节点，求出最低的消耗方案

1. j = set of join nodes
2. for (i in 1...|j|):
3.     for s in {all length i subsets of j}
4.       bestPlan = {}
5.       for s' in {all length d-1 subsets of s}
6.            subplan = optjoin(s')
7.            plan = best way to join (s-s') to subplan
8.            if (cost(plan) < cost(bestPlan))
9.               bestPlan = plan
10.      optjoin(s) = bestPlan
11. return optjoin(j)
