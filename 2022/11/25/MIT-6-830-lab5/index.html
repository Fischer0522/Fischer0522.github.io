index 相关的类

不得不说 B+ 树的实现复杂难度确实是有点超出我的想象，先简单看一看提供的类，再着手完成代码

在 JDK 源码中红黑树的实现方式，一个节点内部存储的为其子节点和父节点的引用，通过引用可以直接获得到节点，由于 HashMap 本身并不会占用太多的空间，因此可以将其全权交给内存，将整棵红黑树放在内存中管理，一个树节点的代码如下：

static final class Entry<K,V> implements Map.Entry<K,V> {
    K key;
  
    V value;
  
    Entry<K,V> left;
  
    Entry<K,V> right;
  
    Entry<K,V> parent;
    //红黑树中的节点默认为黑色
    boolean color = BLACK;
}

而对于 DBMS 中的 B+ 树，如果涉及到了一个大表，想要对他建立索引，节点数可能会有十几万百万之多，并且 DBMS 在同一时刻内可能需要对多张表进行读写，无法在 buffer pool 当中维护一整棵 B+ 树，因此只能使用一种较为松散的组织方式：

  • 一个节点对应一个 Page，在一个 Page 当中存在多个 Entry

  • 在一个 Entry 中，并不维护子节点的引用，只维护子节点的 PageId，想要得到子节点的 Page 时，再通过 getPage ​方法来获取到节点

  • 即只在 buffer pool 当中维护一些节点，当需要其他节点时，再去硬盘中读取获得

BTreePage

继承自 Page 类，同样为接口类型

其中的 parent 存储父节点的 pageNumber

PageTreeId

继承自 PageId，与 HeapPageId 相对应

pacateg 成员变量表示该节点的类型：

  • 0：根节点
  • 1：内部节点
  • 2：叶子节点
  • 3：HeaderPage：专门存储 File 中的 Header bitMap 来表示 page 的使用情况

BTreeRootPtrPage

在数据库中的一个索引即为一颗 B+ 树，BTreeRootPtrPage 存储指向根节点的引用

对于 root 和 header 将其从字节数组转换为 int 类型进行存储，写入和读取时都需要进行 int 和字节数组的转换

BTreeLeafPage

叶子节点，真实存储数据，和 HeapFilePage 对应

拥有 HeapFilePage 基本一致的数据结构类型，header tuples numslots 等

作为叶子节点，还有指向左叶子节点和右叶子节点，存储其 pgNo

BTreeInternalPage

内部节点

Field 存储该 page 中的值

children 数组存储子节点的 paNo

rootPage 本身也为一个特殊的内部节点，物理存储上借助 BTreeInternalPage，使用 BTreeRootPtrPage 进行标识

BTreeHeaderPage

只存储 header

File

对应 HeapFile，用于对磁盘进行写入和读取

private BTreeLeafPage findLeafPage(TransactionId tid, Map<PageId, Page> dirtypages, BTreePageId pid, Permissions perm,
                                      Field f)
				throws DbException, TransactionAbortedException {
	int type = pid.pgcateg();
	if (type == BTreePageId.LEAF) {
		return (BTreeLeafPage) getPage(tid,dirtypages,pid,perm);
	}
	BTreeInternalPage bTreeInternalPage = (BTreeInternalPage) getPage(tid,dirtypages,pid,Permissions.READ_ONLY);
	Iterator<BTreeEntry> iterator = bTreeInternalPage.iterator();
	BTreeEntry entry = null;
	while (iterator.hasNext()) {
		entry = iterator.next();
		if (f == null) {
			return findLeafPage(tid,dirtypages, entry.getLeftChild(), perm,f);
		} else if (entry.getKey().compare(Op.GREATER_THAN_OR_EQ,f)) {
			return findLeafPage(tid,dirtypages,entry.getLeftChild(),perm,f);
		}
	}
	return findLeafPage(tid,dirtypages, entry.getRightChild(), perm,f);

对 B+ 树进行递归搜索：

  • 如果是叶子节点，则直接通过叶子节点中的 getPage 来获取结果即可

  • 如果为内部节点，则需要通过大小来判断：

    • 遍历当前内部节点的 key，如果目标 key 小于当前 key，则向左递归
    • 反之则继续进行遍历当前叶子节点直至末尾
    • 如果遍历完当前内部节点也没有找到大于目标 key 的值，则证明目标 key 只可能存在于最后一个节点的右节点

例：

当如下的 B+ 树当中：如果要搜索 11：

  1. 从根节点 7 开始遍历 7 节点当中没有 >= 11 的值，因此递归搜索 7 的右节点
  2. 在 7 11 13​​内部节点当中遍历到 11 发现满足：entry.getKey().compare(Op.GREATER_THAN_OR_EQ,f)​​,递归搜索其左子节点
  3. 抵达叶子节点，获取值

public BTreeLeafPage splitLeafPage(TransactionId tid, Map<PageId, Page> dirtypages, BTreeLeafPage page, Field field)
		throws DbException, IOException, TransactionAbortedException {
	BTreeLeafPage newRightPage = (BTreeLeafPage) getEmptyPage(tid, dirtypages, BTreePageId.LEAF);
	int numTuples = page.getNumTuples();
	Iterator<Tuple> iterator = page.reverseIterator();
	// 将一半的节点转移至新节点，逆序遍历进行添加
	for (int i =0; i < numTuples / 2;i++) {
		Tuple next = iterator.next();
		page.deleteTuple(next);
		newRightPage.insertTuple(next);
	}
	// 如果原节点的右侧存在节点oldRightPage，那么就需要将oldRightPage和newRightPage进行连接
	BTreePageId oldRightSiblingId = page.getRightSiblingId();
	BTreeLeafPage oldRightPage = null;
	if (oldRightSiblingId != null) {
		oldRightPage = (BTreeLeafPage) getPage(tid,dirtypages,oldRightSiblingId,Permissions.READ_ONLY);
	}
	if (oldRightPage != null) {
		oldRightPage.setLeftSiblingId(newRightPage.getId());
		newRightPage.setRightSiblingId(oldRightPage.getId());
		dirtypages.put(oldRightSiblingId,oldRightPage);
	}

	// 将原节点和新节点进行连接
	page.setRightSiblingId(newRightPage.getId());
	newRightPage.setLeftSiblingId(page.getId());
	dirtypages.put(page.getId(), page);
	dirtypages.put(newRightPage.getId(),newRightPage);
	// 对于叶子节点的复制，需要将中间节点复制一份到父节点当中
	BTreeInternalPage parent = getParentWithEmptySlots(tid, dirtypages, page.getParentId(), field);
	Field mid = newRightPage.iterator().next().getField(keyField);
	BTreeEntry entry = new BTreeEntry(mid, page.getId(), newRightPage.getId());
	parent.insertEntry(entry);
	dirtypages.put(parent.getId(),parent);
	// 更新父节点的指针，修改指向原本page的指针，添加指向newRightPage的指针
	updateParentPointers(tid,dirtypages,parent);
	// 分裂节点用于在insertTuple时使用，大于中间节点则位于新的叶子节点内
	if (field.compare(Op.GREATER_THAN_OR_EQ,mid)) {
		return newRightPage;
	}
	return page;


}

lab 已经给提供了相当丰富的支持，整个过程只需要调用其提供的方法即可，在上述的实现过程当中，简单看一看 updateParentPointers​

该方法用于更新一个内部节点中的所有的 entry，以上面的 9 11 13 ​节点为例，该节点存在四个指针，指向其子节点，自左向右的前三个指针视为前三个节点的左子指针，最后一个指针视为最后一个节点 13 ​的右指针，因此在进行更新时：

  • 首先遍历所有的节点改变其子节点
  • 之后改变最后一个节点的右指针

通过调用 updateParentPointer ​方法分别传入父节点其左右子节点

private void updateParentPointers(TransactionId tid, Map<PageId, Page> dirtypages, BTreeInternalPage page)
		throws DbException, TransactionAbortedException{
	Iterator<BTreeEntry> it = page.iterator();
	BTreePageId pid = page.getId();
	BTreeEntry e = null;
	while(it.hasNext()) {
		e = it.next();
		updateParentPointer(tid, dirtypages, pid, e.getLeftChild());
	}
	if(e != null) {
		updateParentPointer(tid, dirtypages, pid, e.getRightChild());
	}
}

在 updateParentPointer ​当中，获取到子节点、父节点后，由于父节点已经指向了子节点，因此只需要通过子节点的 setParentId ​设置一下父节点即可

private void updateParentPointer(TransactionId tid, Map<PageId, Page> dirtypages, BTreePageId pid, BTreePageId child)
		throws DbException, TransactionAbortedException {

	BTreePage p = (BTreePage) getPage(tid, dirtypages, child, Permissions.READ_ONLY);

	if(!p.getParentId().equals(pid)) {
		p = (BTreePage) getPage(tid, dirtypages, child, Permissions.READ_WRITE);
		p.setParentId(pid);
	}

}

为了维护 B+ 树的结构，在 insert 和 delete 时会涉及到节点之间的分裂、合并、以及 tuple 的窃取。共涉及到以下五个方法，比较难处理的为对父节点重新设置字节点，和父节点的移动，其他子节点的移动较为简单，只需要通过迭代器来操作即可。

统一说一下父节点的处理方式

叶子节点的窃取

由于窃取的为叶子节点，因此其父节点存在叶子节点对应的entry，无需新增或者删除，首先先进行子节点的窃取，之后再更改父节点中的值即可（取窃取后的右节点的第一个值）

entry.setKey(mid.getField(keyField));
parent.updateEntry(entry);

中间节点的窃取

相比于叶子节点，中间节点就相对较为复杂，由于子节点中entry的数量不变，父节点也只是进行调整，在数量上也不会改变，以从右侧进行窃取为例：将父节点中的作为第一个添加到左侧节点中的entry，右侧的最后一个节点添加到父节点当中

public void stealFromLeftInternalPage(TransactionId tid, Map<PageId, Page> dirtypages,
		BTreeInternalPage page, BTreeInternalPage leftSibling, BTreeInternalPage parent,
		BTreeEntry parentEntry) throws DbException, TransactionAbortedException {
	// some code goes here
       // Move some of the entries from the left sibling to the page so
	// that the entries are evenly distributed. Be sure to update
	// the corresponding parent entry. Be sure to update the parent
	// pointers of all children in the entries that were moved.
	//计数
	int currentNumEntries = page.getNumEntries();
	int siblingNumEntries = leftSibling.getNumEntries();
	int targetNumEntries = (currentNumEntries + siblingNumEntries) / 2;

	Iterator<BTreeEntry> iterator = leftSibling.reverseIterator();
	BTreeEntry entry = iterator.next();

	// 先将中间节点移动到右节点
	BTreeEntry mid = new BTreeEntry(parentEntry.getKey(),entry.getRightChild(),page.iterator().next().getLeftChild());
	page.insertEntry(mid);
	currentNumEntries++;
	// 从左侧兄弟节点进行窃取

		leftSibling.deleteKeyAndRightChild(entry);
		page.insertEntry(entry);

		currentNumEntries ++;

	leftSibling.deleteKeyAndRightChild(entry);
	parentEntry.setKey(entry.getKey());
	parent.updateEntry(parentEntry);

	dirtypages.put(page.getId(), page);
	dirtypages.put(leftSibling.getId(), leftSibling);
	dirtypages.put(parent.getId(), parent);
	updateParentPointers(tid,dirtypages,page);


}
